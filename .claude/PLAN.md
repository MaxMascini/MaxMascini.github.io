# Plan: Replace SRTM with NRCan HRDEM (LiDAR, 1m source)

## Context

The current topo background uses AWS SRTM data (30m resolution, radar-derived).
NRCan's HRDEM dataset provides 1m LiDAR data for Halifax with proper water body
NoData masking — giving much sharper coastlines and more detailed terrain.

The user will download and clip the data in QGIS, then drop the file in the repo.
The Python script is rewritten to read that local GeoTIFF instead of downloading
from AWS.

---

## QGIS Export Instructions (user does this)

1. Download the Halifax HRDEM DTM GeoTIFF from NRCan FTP (see "Finding the file" below)

2. In QGIS:
   - Load the DTM tile(s) covering Halifax
   - **Raster > Extraction > Clip Raster by Extent**
     Extent: `Xmin -63.76 / Xmax -63.33 / Ymin 44.50 / Ymax 44.76`
   - **Raster > Projections > Warp (Reproject)**
     Target CRS: `EPSG:4326 (WGS84)`
     Resampling: Bilinear
     **Output resolution: 0.0002° (~20m)** — do NOT export at native 1m (~400MB)
   - **Save As**: GeoTIFF, filename: `halifax-hrdem.tif`

3. Place the file at: `scripts/halifax-hrdem.tif`
   (gitignored — only the processed JSON gets committed)

---

## Script Changes: `scripts/fetch-halifax-elevation.py`

Replace the AWS download + SRTM binary parsing with local GeoTIFF reading via rasterio.

### New flow

```python
import rasterio
import numpy as np
import json, os

TIFF_PATH = os.path.join(os.path.dirname(__file__), 'halifax-hrdem.tif')

with rasterio.open(TIFF_PATH) as src:
    data = src.read(1).astype(float)
    nodata = src.nodata          # e.g. -9999 or -32767
    bounds = src.bounds          # (left, bottom, right, top) in WGS84

if nodata is not None:
    data[data == nodata] = 0
data[data < 0] = 0              # any remaining negatives = water

BOUNDS = {
    "latMin": bounds.bottom,
    "latMax": bounds.top,
    "lonMin": bounds.left,
    "lonMax": bounds.right,
}
```

Then downsample to GRID_W×GRID_H (140×88), normalize (ocean=−1.0, land=[0,1]),
and write JSON — same as current code.
